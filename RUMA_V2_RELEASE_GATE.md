# RUMA V2 Release Gate

This file freezes the practical public-release judgment for the `RUMA V2` architecture-family package.

## Release Framing

The public `V2` release should be framed this way:

`RUMA V2` is an interleaved routed updatable memory architecture family for explicit updates, retained guidance, stale-memory suppression, and grounded evidence control.

That framing is intentionally narrower than a flagship standalone-model claim. The `V2` package is not the place to claim:

- a broad standalone foundation model
- a permanently settled backbone comparison
- a general solution to hallucination
- a universal replacement for standard language-model backbones

## What Is Frozen

- main final-form interleaved `RUMA` architecture story
- standalone structured stabilization suite
- official edit-task package
- broader grounded verification and multi-hop package
- first serious ablations
- broader standalone text-maturity suite
- paper-facing CSV and markdown asset bundle
- final V2 manuscript candidate in [PREPRINT_V2.md](./PREPRINT_V2.md)

## What The Current Evidence Supports

The repo can now honestly support claims like:

- `RUMA` is a serious interleaved architecture for explicit updates, retained guidance, and stale-memory suppression
- `RUMA` remains strong on official edit tasks and bounded standalone stabilization tasks
- the broader controller-based package is strong on `FEVER`, `HotpotQA`, external versioned documents, and the bounded `BEIR` subset bridge
- standalone broader text behavior is no longer missing entirely, and the comparison backbone currently improves that path over the public reference backbone

## What The Current Evidence Does Not Yet Support

The repo should still avoid claiming:

- that `RUMA V2` is already a broad standalone foundation model
- that standalone natural-language calibration is fully mature
- that the backbone question is permanently closed
- that `RUMA` solves hallucination generally

## Must-Have For A Public V2 Release

- [x] frozen main architecture story
- [x] frozen benchmark and evaluation package
- [x] first serious ablations
- [x] final release claim wording locked
- [x] final manuscript wording hardened end-to-end
- [x] public-facing README and release notes prepared
- [x] public package contents and exclusions frozen
- [x] public-facing code map prepared

## Relationship To Later Standalone Work

The repo also contains a larger ongoing standalone branch. That later branch should not block the `V2` release.

The important release fact is simple:

- the current `V2` package is strong enough to stand on its own as an architecture-family release
- broader standalone scaling remains future work rather than a prerequisite for publishing `V2`

## Release Judgment

`RUMA V2` is now release-ready **with an architecture-family framing**.

That means:

- the code package is strong enough
- the manuscript is strong enough
- the benchmark package is strong enough
- the claim boundary is narrow enough to be honest

What remains after release is not "finish V2." It is:

- continue the separate larger standalone branch
- improve broader standalone natural-language capability
- revisit the backbone story only if later evidence materially changes it

That is the release-safe statement this repo should use publicly.
